Getting started

From example to a working narrow-casting screen in a few steps.

Try it right away (demo mode)

The examples work immediately, without an i-Reserve account: while baseUrl in config.js is still set to the placeholder YOURENVIRONMENT, the SDK automatically shows realistic demo data (with times around "now"). Open an example and you see a working screen live — the bottom shows DEMO.

Want to reproduce the real network/CORS path? Start the bundled mini-server (no dependencies): node demo/server.js and open http://localhost:8080/.

Connect an example to your environment

  1. First arrange access: create a read-only API user and allow CORS in the admin panel — see Authentication & CORS.
  2. Pick an example and open its config.js. Fill in:
    • baseUrl → https://YOURENVIRONMENT.i-reserve.net
    • auth → username + password of the API user
    • search → the filter (which products / which period)
    • optionally columns and columnMap if your fields are named differently
  3. Open that example's index.html on the screen. As soon as a real baseUrl is filled in, the SDK automatically switches from demo to live data.

Customise the styling

All colours, shapes and the font are variables at the top of shared/ireserve.css (:root { --ir-primary: … }). Adjust those for the i-Reserve or client styling; the whole SDK follows automatically. Replace shared/i-reserve-logo.png with the desired logo.

Shipping a single example

To hand a single example to a client, copy the example folder and the shared/ folder together (the examples reference ../../shared/ relatively). Keep the relative folder structure intact.

Security: with narrow casting the API credentials are readable in the browser. Always use a read-only API user with access to only the required products. See the security checklist.
